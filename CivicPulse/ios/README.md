# CivicPulse iOS

A SwiftUI iPhone app (iOS 17+) for reading admin-curated AI-policy news and taking civic action: messaging your members of Congress, registering to vote, and supporting pro-AI-safety candidates.

## Requirements

- macOS with Xcode 15 or later
- [XcodeGen](https://github.com/yonaskolb/XcodeGen): `brew install xcodegen`
- The CivicPulse backend running locally (defaults to `http://localhost:8000`)

## Generate the Xcode project

The Xcode project is generated from `project.yml`; no `.xcodeproj` is checked in.

```bash
cd ios
xcodegen generate
open CivicPulse.xcodeproj
```

Re-run `xcodegen generate` whenever you add, remove, or rename source files.

## Configure the backend URL

The API base URL lives in `CivicPulse/Config.swift`:

```swift
static let baseURL = URL(string: "http://localhost:8000")!
```

- **Simulator:** `http://localhost:8000` works as-is (the simulator shares your Mac's loopback interface).
- **Physical device:** change it to your Mac's LAN IP, e.g. `http://192.168.1.20:8000`, and make sure the phone is on the same network. The generated `Info.plist` already sets `NSAllowsLocalNetworking` so plain-HTTP local traffic is permitted during development.

## Run on the simulator

1. Start the backend (e.g. `uvicorn app:app --port 8000` from the backend directory).
2. In Xcode, select the **CivicPulse** scheme and an iPhone simulator (e.g. iPhone 15).
3. Press **Cmd-R**. On first launch you'll be asked for a ZIP code and (optionally) a name and email, then shown your representatives.

Or from the command line:

```bash
xcodebuild -project CivicPulse.xcodeproj -scheme CivicPulse \
  -destination 'platform=iOS Simulator,name=iPhone 15' build
```

## Push notifications

The app requests notification permission at launch, registers for remote notifications, and POSTs the hex APNs device token to `POST /api/users/{id}/device-token`. To make pushes actually arrive:

1. **Apple Developer setup**
   - In Xcode → Signing & Capabilities, select your development team. The Push Notifications capability is already wired up via `CivicPulse/CivicPulse.entitlements` (`aps-environment: development`).
   - In the [Apple Developer portal](https://developer.apple.com/account), create an **APNs Auth Key** (Keys → "+" → enable Apple Push Notifications service). Download the `.p8` file and note the **Key ID** and your **Team ID**.
2. **Backend setup**
   - Give the backend the `.p8` key, Key ID, Team ID, and the bundle ID `org.techoversight.civicpulse`, and have it send to the APNs **sandbox** environment (`api.sandbox.push.apple.com`) for development builds.
3. **Device required**
   - Remote push requires a physical device (the simulator does not receive real APNs pushes, though `xcrun simctl push` can simulate them for UI testing).

For App Store / TestFlight builds, switch the entitlement to `aps-environment: production` and point the backend at the production APNs host.
